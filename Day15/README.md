# Day 15 – Understanding Attack Surface

## 1. Three attack surfaces I identified on my system

### 1. Cracked software
One attack surface on my system is cracked software or programs from unofficial websites.

**Why it is risky:**  
These files can be unsafe and may contain malware even if they look normal or work correctly.

### 2. Turning off antivirus
Another attack surface is turning off antivirus or firewall to install or run some programs.

**Why it is risky:**  
When protection is off, the system becomes more vulnerable and harmful files may run without being detected.

### 3. Browser extensions
Browser extensions can also be an attack surface.

**Why it is risky:**  
Some extensions ask for many permissions, and if they are unsafe, they can collect data or redirect me to unsafe websites.

### 4. Open ports and running services
Open ports and background services can also increase the attack surface.

**Why it is risky:**  
If a service is running and has a weakness, an attacker may try to use it to access the system.


## 2. One thing I realized about my own system security

I realized that some simple things I do for convenience can actually make my system less secure.

For example, using cracked software or turning off antivirus may help something work faster, but it can also create bigger security risks.


## 3. One step that could reduce the attack surface

One step I can take is to stop using software that needs me to disable antivirus.

Instead, I should use trusted software and keep my protection turned on.
